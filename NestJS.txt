6. 🔗 integrations/
-------------------------------------------------------------
6.1 billing/
-------------------------------------------------------------
integrations/billing/
├── billing.module.ts
├── billing.service.ts
├── billing-provider.interface.ts
├── providers/
│   ├── fossbilling.provider.ts
│   ├── billmanager.provider.ts
Ответственность:
отправка данных обратно в billing

Billing Provider candidates:
- FossBilling + custom YooKassa payment adapter
- BILLmanager / другой provider с готовой YooKassa интеграцией

❗ Core logic зависит от BillingProvider interface, а не от конкретного продукта.
❗ Custom YooKassa adapter для FossBilling живёт на стороне billing и не является частью core Orchestrator.
-------------------------------------------------------------
6.2 vpn/
-------------------------------------------------------------
integrations/vpn/
├── vpn.module.ts
├── vpn.service.ts
├── vpn.client.ts
Назначение:
удалённое управление 3x-ui нодами напрямую через API панели.
Отдельный агент / adapter service на VPN ноды не устанавливается.
Provider switch:
- VPN_PROVIDER=noop by default
- VPN_PROVIDER=3x-ui enables direct 3x-ui HTTP adapter
- 3x-ui adapter uses /login session auth and /panel/api/inbounds client endpoints
- vpn_nodes.apiKey contains {"username":"...","password":"..."} or username:password
-------------------------------------------------------------
