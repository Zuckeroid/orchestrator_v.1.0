Admin UI v1.0
-------------

Location:
- orchestrator-main/admin-ui

Stack:
- React
- Vite
- TypeScript

Development:
- Backend API: http://localhost:3000/api/v1
- Admin UI: http://localhost:5173
- Vite proxies /api/v1 to http://localhost:3000

Commands from orchestrator-main:
- npm run admin:dev
- npm run admin:build

Login/settings:
- API base URL default: /api/v1
- Admin key is stored in browser localStorage.
- Actor is sent as X-Admin-Actor.

Implemented screens:
- Dashboard
- Plan Mapping
- VPN Nodes
- Storage Backends
- Provisions
- Webhook Tester
- Events
- Audit

Implemented actions:
- add plan mapping
- edit plan mapping
- delete unused plan mapping
- add VPN node
- disable VPN node
- add storage backend
- disable storage backend
- delete provision now
- send signed billing webhook test events
- refresh dashboard/state

Plan Mapping:
- Billing creates commercial tariffs/products: prices, periods, invoices, payments, renewals.
- Orchestrator stores technical mapping for those tariffs.
- Billing plan ID maps webhook plan/product IDs to Orchestrator limits.
- Mapping controls VPN enabled, Storage enabled, max devices/IP limit, and storage bytes.
- Billing plan ID is set on creation and is not edited from Admin UI.
- Mapping deletion is blocked when provision records already reference it.

Storage Backends:
- Admin UI manages S3-compatible storage backend records for Orchestrator.
- Supported v1.0 fields: provider, endpoint, region, access key, secret key, bucket prefix, capacity.
- Disable action marks backend inactive for new provisions without deleting historical records.
- Real storage provider can be MinIO or S3-compatible service; noop/mock path remains valid for local checks.

Backend integration:
- Uses X-Admin-Api-Key
- Uses X-Admin-Actor
- Uses X-Request-Id per request
- Backend CORS allows ADMIN_UI_ORIGIN, default http://localhost:5173

Webhook Tester:
- Sends requests to /api/v1/webhook/billing.
- Uses WEBHOOK_API_KEY as x-api-key.
- Uses WEBHOOK_SIGNING_SECRET to create x-signature.
- Uses x-timestamp and HMAC SHA-256 over the raw JSON body.
- Same eventId can be sent twice to verify idempotency.
- Webhook key and signing secret are stored in browser localStorage for local/operator use.

v1.0 scope:
- Operator control panel for the existing Admin API.
- No multi-user RBAC.
- No billing cockpit.
- No storage file browser.
- No node migration UI.
