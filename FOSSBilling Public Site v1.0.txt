FOSSBilling Public Site v1.0
============================

Цель
----
Подготовить публичную сторону FOSSBilling для модерации YooKassa и первых продаж.

Публичное позиционирование v1.0:
- продаем Storage;
- сервис позиционируется как облачное хранилище личной информации для обычных граждан;
- не используем на публичном сайте слова VPN, proxy, 3x-ui, inbound, node, tunneling;
- услугу описываем как облачное хранение файлов и защищенный доступ к файлам;
- количество устройств в тарифе описываем как лимит защищенного доступа к файлам.

Главная задача сайта:
- регистрация пользователя;
- выбор одного из трех тарифов;
- прием оплаты;
- отправка инструкций на зарегистрированный email.

Пользовательский интерфейс не перегружаем:
- публично: короткое описание сервиса, тарифы, вход/регистрация, обязательные юридические страницы;
- в аккаунте: три тарифа, счета, статус услуги, минимальная справочная информация;
- подробные инструкции по настройке и доступу отправляются на email после оплаты.

Техническая реальность Orchestrator не меняется:
- billing product -> externalPlanId;
- externalPlanId -> Plan Mapping;
- Plan Mapping может включать storageEnabled и технические параметры доступа;
- Orchestrator не раскрывает пользователю внутреннюю инфраструктуру.

Обязательные публичные страницы
-------------------------------
1. Главная
   - Что продаем: облачное хранилище личной информации.
   - Для кого: обычные пользователи, которым нужно хранить личные файлы, документы, фото и важные материалы.
   - Основной CTA: выбрать тариф.
   - Без технических слов про VPN/3x-ui.

2. Тарифы
   - Реальные тарифы с ценами.
   - Объем хранилища.
   - Количество устройств для защищенного доступа.
   - Срок действия тарифа.
   - Что происходит после оплаты.

3. Как получить доступ
   - Клиент регистрируется и оплачивает тариф.
   - После оплаты в личном кабинете появляются данные доступа к хранилищу.
   - Доступ активируется автоматически, обычно в течение нескольких минут.
   - При проблемах клиент обращается в поддержку.

4. Пользовательское соглашение / оферта
   - Предмет услуги: предоставление доступа к облачному файловому хранилищу.
   - Условия оплаты и продления.
   - Правила использования.
   - Ограничения ответственности.
   - Условия блокировки при нарушениях.

5. Политика конфиденциальности
   - Какие данные собираются.
   - Для чего используются email, платежные данные, технические логи.
   - Как пользователь может обратиться по вопросам персональных данных.

6. Возвраты
   - Условия возврата.
   - Срок обработки обращения.
   - Канал обращения в поддержку.
   - Важно: точный текст должен соответствовать юрлицу/ИП и реальной политике возвратов.

7. Контакты и реквизиты
   - Название юрлица или ИП.
   - ИНН.
   - ОГРН/ОГРНИП.
   - Юридический адрес.
   - Email поддержки.
   - Телефон.
   - Режим обработки обращений.

8. Личный кабинет
   - Регистрация.
   - Авторизация.
   - Три тарифа на выбор.
   - Заказы/счета.
   - Статус услуги.
   - Сообщение, что инструкции отправляются на зарегистрированный email после оплаты.

Формулировки для карточек тарифов
---------------------------------
Хорошо:
- "Защищенный доступ к файлам для 1 устройства"
- "Личное облачное хранилище"
- "Доступ к файлам после оплаты"
- "Хранение документов, фото и рабочих файлов"
- "Поддержка по email"

Не использовать публично:
- VPN
- VLESS
- Xray
- 3x-ui
- inbound
- node
- proxy
- обход блокировок
- анонимность
- смена IP
- трафик через сервер

Страница тарифов: структура
---------------------------
Для каждого тарифа:
- название;
- цена в рублях;
- период оплаты;
- объем хранилища;
- количество устройств для защищенного доступа;
- срок активации;
- кнопка "Заказать".

Текст под тарифами:
После оплаты заказ обрабатывается автоматически. Данные доступа к хранилищу появляются в личном кабинете. Если доступ не появился в течение нескольких минут, напишите в поддержку.

Страница "Как получить доступ": черновик
---------------------------------------
После оплаты тарифа система автоматически создает доступ к вашему файловому хранилищу. Данные доступа отображаются в личном кабинете в карточке услуги.

Если вы сменили устройство или не можете открыть хранилище, обратитесь в поддержку. Мы проверим статус услуги и поможем восстановить доступ.

Технические требования перед отправкой в YooKassa
------------------------------------------------
- Сайт открывается публично.
- Работает HTTPS.
- Нет заглушек "в разработке".
- Нет тестовых товаров и тестовых цен.
- Есть реальные тарифы.
- Есть описание способа получения услуги после оплаты.
- Есть оферта или пользовательское соглашение.
- Есть политика конфиденциальности.
- Есть контакты и реквизиты.
- Есть рабочая регистрация/авторизация клиента.
- Клиент может сформировать заказ/счет.
- После оплаты понятно, где клиент получает услугу.

FOSSBilling mapping
-------------------
FOSSBilling product names должны быть публичными:
- Storage Start
- Storage Plus
- Storage Family

Orchestrator externalPlanId может оставаться техническим:
- storage_start
- storage_plus
- storage_family

Пример product_plan_map_json:
{
  "1": "storage_start",
  "2": "storage_plus",
  "3": "storage_family"
}

Тарифы v1.0:
- Storage Start: 1 устройство, 290 руб./мес.
- Storage Plus: 3 устройства, 790 руб./мес.
- Storage Family: 5 устройств, цена уточняется перед публикацией.

Публичные названия и внутренние externalPlanId не обязаны совпадать.

v1.0 решение
------------
Для модерации и запуска платежей в v1.0 сайт позиционируется как сервис облачного хранения файлов.
VPN/3x-ui часть не является публичным продуктовым обещанием и не описывается на сайте.
