# DPI Monitor Plan

Цель: определять, почему у пользователя не работает конкретная VPN-нода, и автоматически уводить его на рабочий маршрут без сбора посещенных сайтов и содержимого трафика.

## Что собираем

Только технические события диагностики:

- версия приложения и платформа;
- тип сети: `wifi`, `cellular`, `unknown`;
- оператор: `carrierName`, `mcc`, `mnc`;
- нода: `nodeId`, `nodeName`, `nodeCountry`, `nodeHost`, `nodePort`;
- профиль: `protocol`, `transport`;
- тип проверки: `dns_resolution`, `node_tcp_connect`, `vpn_handshake`, `tunnel_http`, `route_probe`;
- результат: `success`, `failed`, `timeout`, `skipped`;
- задержка, код ошибки, короткое техническое сообщение;
- дополнительные детали без пользовательских URL.

Не собираем:

- сайты пользователя;
- DNS-имена, которые пользователь открывал;
- payload трафика;
- персональные данные сверх обезличенного `installIdHash`.

## Классификации

- `dns_suspected` - домен/служебный DNS не резолвится или возвращает некорректный ответ.
- `ip_or_port_suspected` - TCP до IP/порта ноды не проходит.
- `protocol_suspected` - порт доступен, но VPN handshake не проходит или рвется.
- `node_exit_degraded` - VPN поднялся, но контрольный HTTP через туннель не отвечает.
- `provider_dpi_suspected` - одна и та же нода массово падает у конкретного оператора, но работает из других сетей.
- `client_network_changed` - Android/Chrome сменил сеть во время теста.
- `unknown` - данных мало, нужна повторная проверка.

## Этапы

1. Оркестратор хранит сырые события и строит сводку по `node + carrier + classification`.
2. Billing/appbridge принимает события от Android с device token и прокидывает их в оркестратор.
3. Android выполняет диагностику при ошибке подключения: DNS, TCP к ноде, VPN handshake, контрольный HTTP через туннель.
4. Оркестратор помечает проблемные сочетания оператор/нода и отдает приложению рекомендации в конфиге.
5. Android автоматически предлагает или применяет fallback-ноду, если выбранная нода подозрительно часто недоступна у текущего оператора.

## Пользовательский сценарий

Пользователь не должен видеть слова DPI. В интерфейсе достаточно:

> Сервер недоступен у вашего провайдера. Переключаем на рабочий.

Для диагностического экрана можно показывать:

- Нода недоступна;
- Похоже, провайдер блокирует маршрут;
- Найден рабочий сервер: Москва / другой регион.
